# KinoFinder — плеер для IMDb и Кинопоиска

Расширение для Chrome (Manifest V3). На странице фильма/сериала на **IMDb**
(`imdb.com/title/ttXXXXXXX`) или **Кинопоиске** (`kinopoisk.ru/film/...`,
`kinopoisk.ru/series/...`) добавляет плавающую кнопку **«Смотреть онлайн»**.

При клике открывается отдельное окно расширения, в котором **параллельно**
запускается загрузка двух плееров:

- `kinoboxtv.tatnet.app`
- `kinoplayertop.tatnet.app`

Показывается тот источник, который **первым успешно отдал контент**
(событие `load` у iframe). Второй остаётся загруженным в фоне — можно в любой
момент переключиться на него вручную кнопкой в шапке окна.

## Установка (режим разработчика)

1. Открой `chrome://extensions`.
2. Включи переключатель **«Режим разработчика»** (правый верхний угол).
3. Нажми **«Загрузить распакованное расширение»**.
4. Укажи папку `kinofinder` (эту).
5. Зайди на любую страницу тайтла на IMDb или Кинопоиске — внизу справа
   появится кнопка «Смотреть онлайн».

## Структура проекта

```
kinofinder/
├── manifest.json        # манифест MV3
├── background.js        # service worker — открывает окно плеера
├── content_imdb.js       # кнопка + извлечение tt-id и названия на IMDb
├── content_kp.js          # кнопка + извлечение id и названия на Кинопоиске
├── content.css            # стили плавающей кнопки
├── player.html             # окно плеера
├── player.css              # стили окна плеера
├── player.js                # гонка двух источников, переключение
└── icons/                   # иконки 16/48/128
```

## Как определяется фильм

- **IMDb**: id берётся из URL (`tt1234567`), название — из `og:title`,
  год — из ссылки `releaseinfo` или из `<title>`.
- **Кинопоиск**: id берётся из URL (`/film/123456/` или `/series/123456/`),
  название и год — из `og:title` и блока с годом на странице.

Эти данные передаются в `player.html` через query-параметры
(`imdb_id`, `kp_id`, `title`, `year`), а оттуда — в обе ссылки плееров.

## ⚠️ Важно: проверь параметры embed-URL плееров

Я не знаю точного формата query-параметров, которые принимают
`kinoboxtv.tatnet.app` и `kinoplayertop.tatnet.app` (это твои собственные
сервисы/прокси, и их API мне недоступен). Сейчас в `player.js` используется
разумный шаблон по аналогии с твоими прошлыми проектами (KinoBox/KinoBD
players, IMDb/КП id в query):

```js
// kinoboxtv.tatnet.app
?imdb_id=ttXXXXXXX        // или
?kp_id=123456

// kinoplayertop.tatnet.app
?imdb_id=ttXXXXXXX        // или
?kinopoisk_id=123456
```

Если реальные сервисы ждут другие имена параметров (например `id=`, `kp=`,
хэш-роутинг `#/imdb/tt123` и т.п.) — поменяй это в одном месте, в файле
`player.js`, в объекте `SOURCES`, в функциях `buildUrl`. Больше нигде менять
ничего не нужно, вся остальная логика (гонка/переключение/отображение)
работает независимо от конкретного формата URL.

```js
const SOURCES = {
  kinobox: {
    buildUrl: ({ imdbId, kpId, title, year }) => {
      // <-- редактируй здесь
    },
  },
  kinoplayertop: {
    buildUrl: ({ imdbId, kpId, title, year }) => {
      // <-- редактируй здесь
    },
  },
};
```

## Логика "гонки" источников

1. Оба `<iframe>` в `player.html` получают `src` практически одновременно.
2. На каждый навешан обработчик `load`. Браузер вызывает его, когда документ
   внутри iframe полностью загрузился (включая ошибочные/пустые страницы —
   ограничение кросс-доменных iframe: точнее определить "есть фильм или нет"
   без доступа к контенту iframe нельзя).
3. Кто первый вызвал `load` — тот и показывается, статус-экран со спиннером
   скрывается.
4. Кнопки источников в шапке становятся активными по мере готовности каждого
   фрейма — можно переключиться вручную в любой момент.
5. Если за 9 секунд (`RACE_TIMEOUT_MS`) ни один источник не загрузился —
   показывается тот, что всё-таки успел (если есть), либо сообщение с
   просьбой переключиться вручную.

## Доработки, которые можно добавить позже

- Кэширование последнего использованного источника в `chrome.storage`, чтобы
  при следующем фильме сразу пробовать тот, что чаще "выигрывает".
- Опциональная страница настроек (popup) с переключателем
  "всегда KinoBox / всегда KinoPlayer / авто-гонка".
- Поддержка сериалов с выбором сезона/серии (если плееры это поддерживают
  через query-параметры `season`/`episode`).
- Публикация в Chrome Web Store (нужен `key` в manifest и подписанный zip).
